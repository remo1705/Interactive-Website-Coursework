Use of Let:

In apply.js and visit.js, I utilized the `let` keyword. Essentially, `let` lets you declare variables that are scoped to a specific block, such as within a function. Therefore, I chose `let` for variables that were only needed within particular blocks. For instance, I employed `let` in `for` loops to declare the variable `i` in the initialization statement. (The variables 'i'/'index' were used, both used with the same purpose in apply.js).

In information.js, the variable `indexofcurrent` stores the index of the currently displayed message in the `msgloop` function, allowing for smooth cycling through the messages. Using the `let` keyword ensures that `indexofcurrent` is block-scoped, preventing conflicts with other variables and enhancing code clarity and maintainability.

Use of 'const':

`const` serves as an alternative method for declaring a variable. Variables declared with `const` possess two key characteristics: they are block-scoped (similar to `let`), and their values cannot be modified. This is why I prefer using `const` for variables that should remain unchanged. For instance, I declared the `msgs` variable as a `const`, which holds an array of all the messages to be displayed on the program page.
